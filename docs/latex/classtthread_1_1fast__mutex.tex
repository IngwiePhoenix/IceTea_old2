\subsection{Detailed Description}
Fast mutex class. 

This is a mutual exclusion object for synchronizing access to shared memory areas for several threads. It is similar to the \hyperlink{classtthread_1_1mutex}{tthread\+::mutex} class, but instead of using system level functions, it is implemented as an atomic spin lock with very low C\+PU overhead.

The {\ttfamily \hyperlink{classtthread_1_1fast__mutex}{fast\+\_\+mutex}} class is N\+OT compatible with the {\ttfamily \hyperlink{classtthread_1_1condition__variable}{condition\+\_\+variable}} class (however, it IS compatible with the {\ttfamily \hyperlink{classtthread_1_1lock__guard}{lock\+\_\+guard}} class). It should also be noted that the {\ttfamily \hyperlink{classtthread_1_1fast__mutex}{fast\+\_\+mutex}} class typically does not provide as accurate thread scheduling as a the standard {\ttfamily mutex} class does.

Because of the limitations of the class, it should only be used in situations where the mutex needs to be locked/unlocked very frequently.

\begin{DoxyNote}{Note}
The \char`\"{}fast\char`\"{} version of this class relies on inline assembler language, which is currently only supported for 32/64-\/bit Intel x86/\+A\+M\+D64 and Power\+PC architectures on a limited number of compilers (G\+NU g++ and MS Visual C++). For other architectures/compilers, system functions are used instead. 
\end{DoxyNote}
