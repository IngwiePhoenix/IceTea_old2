Thread local storage keyword. 

A variable that is declared with the {\ttfamily thread\+\_\+local} keyword makes the value of the variable local to each thread (known as thread-\/local storage, or T\+LS). Example usage\+: 
\begin{DoxyCode}
1 // This variable is local to each thread.
2 thread\_local int variable;
\end{DoxyCode}
 \begin{DoxyNote}{Note}
The {\ttfamily thread\+\_\+local} keyword is a macro that maps to the corresponding compiler directive (e.\+g. {\ttfamily \+\_\+\+\_\+declspec(thread)}). While the C++11 standard allows for non-\/trivial types (e.\+g. classes with constructors and destructors) to be declared with the {\ttfamily thread\+\_\+local} keyword, most pre-\/\+C++11 compilers only allow for trivial types (e.\+g. {\ttfamily int}). So, to guarantee portable code, only use trivial types for thread local storage. 

This directive is currently not supported on Mac OS X (it will give a compiler error), since compile-\/time T\+LS is not supported in the Mac OS X executable format. Also, some older versions of Min\+GW (before G\+CC 4.\+x) do not support this directive. 
\end{DoxyNote}
