Memory management

Overview

Efficient memory manager.

Symbols for configuring memory manager

  • OS_HEAP_CHUNK_SIZE defines size of chunk where each of objects with size <= OS_HEAP_CHUNK_SIZE is stored. Value must be a power of two. Maximum recommended value is 256. Defaults are 128 or 256 depending on memory allocation strategy (see OS_USE_HEAP_SAVING_MODE).
  • OS_HEAP_PAGE_SIZE is used to define size of page where medium sized objects are stored. Value must be a power of two. Maximum recommended value is 65536. After exhausting existing page, new one with size OS_HEAP_PAGE_SIZE is allocated from system. Default value is 32 kbytes.
  • OS_DEBUG enables debugging mode in allocator. It uses more memory, but enables memory overrun and heap corruption detection.
  • OS_USE_HEAP_SAVING_MODE enables heap-saving mode where pages are not used. But enabling this results in performance drop.
  • DEBUG_APP_HEAP_DUMP_LEAK_ON_EXIT enables creation of log-file named "dump-err-exit.log" on application exit. File contains information about memory leaks.
